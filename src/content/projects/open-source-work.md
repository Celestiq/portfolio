I believe in paying forward the tools that make my work possible. When I find a bug in something I use, I try to fix it upstream rather than work around it locally.

## Notable contributions

The most useful kind of open source contribution is the one nobody notices: fixing the error message that confused you, adding the example that would have saved you an hour, documenting the edge case you ran into.

Most of my contributions are small. I think that is fine. The codebase that accumulates a hundred small improvements over a year is more maintainable than one that received two heroic overhauls.

## What I have learned

Reading other people's code is the fastest way to see how the same problem can be solved in completely different ways — some better than what you would have written, some worse, and occasionally some that make you rethink your assumptions entirely.

Code review culture, commit message hygiene, issue triage — these are things I learned by contributing to projects run by people who take them seriously.
